Gastty v0.7.3

Fixes

  • Ctrl+C / Ctrl+D / Ctrl+W now work inside TUIs like Claude Code, lazygit, and helix. They already worked at the bash/zsh prompt; the bug only hit applications that negotiate the modern kitty keyboard protocol with the terminal. We were stripping the letter from the key event before libghostty could encode the proper CSI u sequence — now we follow upstream Ghostty's input convention exactly.

What's new

  • Confirm before closing a tab or pane with something running. ⌘W (or the tab's X button) now pops a "Close Tab?" sheet when the tab has an SSH session, vim, claude, or any other running process — so a stray keystroke doesn't kill your work. Idle shells still close instantly, so nothing changes for the common case. Splits get the same treatment with a "Close Pane?" sheet scoped to just the focused pane.
