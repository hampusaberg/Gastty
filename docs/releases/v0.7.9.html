Gastty v0.7.9

Fixes

  • SSH connections no longer die silently after idle periods. Saved connections launched from the sidebar, Quick Connect, or the Connections window now pass ServerAliveInterval=60 and ServerAliveCountMax=3 to OpenSSH — a tiny encrypted ping every 60 seconds keeps NAT/firewall mappings open so a left-open SSH session survives lunch, meetings, or overnight without "Write failed: Broken pipe". Industry-standard values (what Warp, SecureCRT, and most modern SSH clients ship by default). Direct sessions, ProxyJump tunnels, and credential-driven sessions all get the same treatment.

If you also launch SSH from the shell directly (ssh foo, git push, scp), add a Host * block to ~/.ssh/config with the same options to get the same protection there.
